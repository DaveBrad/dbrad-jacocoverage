Report brief description

Brief description of the kind of reports that are generated from dbrad-jacocoverage and its settings.

Table of contents

Report kinds

For code coverage there are a number of settings for getting reports, and a location as to where to place them. What kinds of report layout and where they are stored is a users decision.

The reports produced are:

XML report

Produced from a jacocoverage binary coverage data file (internal). This file may be kept and externally processed by the user's testing/coverage post-processes. The format is provided by www/eclemma.org jacocoagent.

Netbeans console report

Prints the report with basic values for packages & classes to the NB console (Output view). There is no file produced for this kind of report. This report kind is useful for quick viewing, but if the project is large then it would be better to get an HTML report.

The result lines are color coded to represent pass, fail states.

The by-project and by-grouped/bundled settings define how the reports are presented, as shown following.

HTML report file(s)

HTML reports are generated via the jacocoagent architecture from the internal and/or XML file. The format of the reports is describe at 'www.eclemma.org', please see this site for details, however, for the most part is unnecessary.

The HTML may be set to automatically launch in the system default browser. Most modern browsers allow a program to launch the browser; but there are exceptions that disallow automatic browser launch capability, or need special protocols (Known: Microsoft Edge).

The by-project and by-grouped/bundled settings define how the reports are presented, as shown following.

By-project (HTML)

This layout presents a header page (created internally by dbrad-jacocoverage) and then each project to report-on is added as a link-item. The display of the added projects report is viewed via a view within the header page.

The project report document is the jacocoverage report folder-file set for a project (its classes and methods).

Browser reporting error condition

There is the potential for a report not being displayed with the following browser display (or similar). This is caused by limitations of browsers and very long paths to a HTML file.

The example below shows dbrad-jacocoverage being coverage tested by itself. As the module has a long name, file paths are also long and result in too long a path for the browser-and-platform file system.

Overcome the issue by:

  1. Copy the report directory to a fewer leading folders path.
  2. Have reports going to a user-defined directory with fewer leading folders.

Fewer leading folders: 

  • file:///tmp_yyyy/clone/dbrad-jacocoverage/dbrad-jacocoverage-suite/dbrad-jacocoverage-plugin/.jacocodbrad/ts_20180127_134734_117_reportDir/............
  • file:///reportDir/ts_20180127_134734_117_reportDir/............

By-grouped/bundled (HTML)

This layout is created by jacocoverage and groups/bundles all the projects and all project-classes & methods into a single report folder-file set.
